Resumen Ejecutivo

La máquina Fawn expone vsftpd 3.0.3 en el puerto 21 con acceso anónimo habilitado. Esto permite que cualquier persona se conecte sin credenciales reales y descargue archivos del servidor. La flag estaba accesible directamente en el directorio raíz del FTP. Hecho notable: el script NSE ftp-anon de Nmap reveló la flag antes de que nos conectáramos manualmente, demostrando que leer el output completo de nmap -sC puede ahorrarnos pasos adicionales.

¿Qué es FTP y por qué el acceso anónimo es peligroso?

FTP (File Transfer Protocol) es un protocolo de red para transferir archivos. Creado en 1971, es uno de los protocolos de Internet más antiguos aún en uso. El acceso anónimo permite conectarse con el usuario anonymous y cualquier contraseña — diseñado para distribución pública de archivos.

⚠ ¿Cuándo es peligroso?

  • Archivos sensibles expuestos: Si los permisos no están bien configurados, archivos privados quedan accesibles a cualquier persona.
  • Texto plano: FTP no cifra el tráfico — contraseñas y archivos viajan en claro, visibles con Wireshark.
  • Sin autenticación real: El servidor acepta cualquier contraseña para el usuario anonymous.

Reconocimiento — Paso a Paso

Paso 1 — Reconocimiento

Verificar conectividad con ping

¿Por qué este comando?

Confirmar que la máquina está activa y obtener una primera pista del OS mediante el TTL, antes de lanzar herramientas más pesadas.

ping -c2 10.129.18.193
ping → herramienta ICMP de diagnóstico de red
-c2 → enviar 2 paquetes y detenerse. Sin este flag corre indefinidamente.
10.129.18.193 → IP del objetivo en la VPN de HTB
Output
64 bytes from 10.129.18.193: icmp_seq=1 ttl=63 time=43.1 ms
64 bytes from 10.129.18.193: icmp_seq=2 ttl=63 time=43.4 ms
2 packets transmitted, 2 received, 0% packet loss
¿Qué nos dice este output?
  • ttl=63 → Linux confirmado (TTL inicial 64, menos 1 salto de red = 63)
  • 0% packet loss → conectividad perfecta con la VPN de HTB
Decisión tomada

Máquina Linux activa. Iniciamos descubrimiento de puertos con nmap.

Paso 2 — Reconocimiento

Descubrir puertos abiertos — Nmap fase rápida

¿Por qué este comando?

Mapear todos los servicios expuestos. Escaneamos los 65535 puertos para no perdernos ningún servicio en puertos no estándar.

nmap -p- --open -n -T4 -sS 10.129.18.193
-p- → todos los 65535 puertos TCP
--open → solo puertos abiertos (los cerrados/filtrados no son vectores inmediatos)
-n → sin resolución DNS (más rápido)
-T4 → velocidad agresiva (escala T0-T5)
-sS → SYN scan: envía SYN sin completar el handshake (más sigiloso)
Output
PORT   STATE SERVICE
21/tcp open  ftp
¿Qué nos dice este output?
  • 1 solo puerto abierto: 21/tcp → FTP (File Transfer Protocol)
  • Superficie de ataque mínima: un único servicio = un único vector
Decisión tomada

FTP en puerto 21. El árbol de decisión Ghost-Ops indica: ejecutar nmap -sC para que el script ftp-anon detecte automáticamente el acceso anónimo.

Paso 3 — Reconocimiento (EL MOMENTO REVELADOR)

Enumerar FTP con scripts NSE — Nmap ya encuentra la flag

¿Por qué esta fase es especialmente importante en Fawn?

El script NSE ftp-anon se conecta automáticamente como usuario anonymous y lista el directorio. En esta máquina, nmap reveló flag.txt ANTES de que nos conectáramos manualmente. Este es el principio central de Ghost-Ops: leer siempre el output completo de nmap -sC antes de ejecutar comandos adicionales.

nmap -p21 -sC -sV -n 10.129.18.193
-p21 → solo el puerto 21. No re-escanear todos los 65535.
-sC → scripts NSE por defecto. Para FTP incluye: ftp-anon (intenta login anónimo y lista directorio), ftp-syst (info del OS vía FTP), ftp-bounce (verifica si es proxy FTP).
-sV → detectar versión exacta del servicio FTP
-n → sin resolución DNS
Output

21/tcp open  ftp     vsftpd 3.0.3
| ftp-anon: Anonymous FTP login allowed (FTP code 230)
|_-rw-r--r--    1 0        0              32 Jun 04  2021 flag.txt
| ftp-syst:

|      Connected to ::ffff:10.10.14.72
|      Logged in as ftp

|      Control connection is plain text
|_     vsFTPd 3.0.7 - secure, fast, stable

Service Info: OS: Unix
¿Qué nos dice cada línea?
  • vsftpd 3.0.3 → Very Secure FTP Daemon, servidor FTP muy común en Linux
  • ftp-anon: Anonymous FTP login allowed (FTP code 230) → LOGIN ANÓNIMO CONFIRMADO. Código 230 = "User logged in, proceed" (RFC 959). El script se conectó como anonymous y el servidor lo aceptó.
  • -rw-r--r-- 1 0 0 32 Jun 04 2021 flag.txt → ¡La flag está aquí! Tamaño 32 bytes = longitud exacta de un hash MD5. Permisos de lectura para todos.
  • plain text → FTP sin cifrado confirmado. Todo el tráfico es legible en la red.
Decisión tomada

El script NSE confirmó anonymous login Y localizó flag.txt. El ataque está esencialmente resuelto — solo falta conectarnos y descargar el archivo.

⭐ Bonus: Lo que Nmap ya sabía antes de que nos conectáramos

Esta máquina ilustra un principio fundamental de Ghost-Ops: los scripts NSE de Nmap pueden hacer el trabajo de reconocimiento completo. El script ftp-anon:

  1. Se conecta automáticamente al servidor FTP con USER anonymous
  2. Verifica si el servidor responde con código 230 (login OK)
  3. Ejecuta LIST para listar el directorio raíz
  4. Reporta todo en la salida de nmap — incluyendo flag.txt

Regla Ghost-Ops: Leer SIEMPRE el output completo de nmap -sC antes de ejecutar comandos adicionales.

Paso 4 — Explotación

Sesión FTP completa — login anonymous y descarga de flag

¿Por qué conectarnos si nmap ya reveló la flag?

Nmap nos confirmó que flag.txt existe y es accesible, pero necesitamos descargar el archivo para leerlo. El cliente ftp nos permite hacer exactamente eso con una sesión interactiva. Usamos -n para control manual del login.

ftp -n 10.129.18.193
ftp → cliente FTP de línea de comandos estándar de Linux
-n → no hacer auto-login usando ~/.netrc (archivo con credenciales guardadas). Con -n tomamos control manual.
10.129.18.193 → IP del servidor FTP objetivo
Sesión FTP completa

Connected to 10.129.18.193.
220 (vsFTPd 3.0.3)

ftp> quote USER anonymous
331 Please specify the password.

ftp> quote PASS anonymous
230 Login successful.

ftp> ls
-rw-r--r-- 1 0 0 32 Jun 04 2021 flag.txt

ftp> get flag.txt
226 Transfer complete. 32 bytes received.

ftp> bye
221 Goodbye.

Decisión tomada

Archivo descargado exitosamente (32 bytes). Leemos la flag con cat flag.txt.

Leer la flag con cat

cat flag.txt
cat → (concatenate) lee y muestra el contenido de un archivo en la terminal. La forma más directa de leer un archivo de texto en Linux.
flag.txt → archivo descargado del servidor FTP en el paso anterior

  • Coincide con el tamaño de 32 bytes que nos informó nmap en el listado del directorio

Lecciones Aprendidas

⚔ Para el Atacante

  • Puerto 21 abierto → siempre ejecutar nmap -sC primero
  • ftp-anon detecta acceso anónimo automáticamente
  • Si ftp-anon dice YES, el ataque está prácticamente resuelto
  • Orden: nmap -sC → ftp -n → get → bye
  • Siempre explorar subdirectorios en FTP (no solo raíz)

🛡 Para el Defensor

  • Deshabilitar FTP anónimo: anonymous_enable=NO
  • Usar SFTP (FTP sobre SSH) — cifrado y seguro
  • Si anonymous es necesario, solo lectura, sin archivos sensibles
  • Revisar logs: /var/log/vsftpd.log
  • Firewall: restringir puerto 21 a IPs específicas
