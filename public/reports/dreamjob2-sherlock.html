DRATzarus — Análisis del RAT

Relación con el Ecosistema MATA

La primera pregunta crítica en análisis de malware es: ¿qué conocemos ya de este artefacto? MITRE ATT&CK (S0694) documenta que DRATzarus comparte similitudes con MATA, el framework modular de Lazarus Group.

Lazarus Group Malware Ecosystem
├── MATA Framework (modular, multi-platform)
└── DRATzarus (RAT derivado, similitudes en código)
├── Torisma (implante de reconocimiento)
└── Otros: BLINDINGCAN, COPPERHEDGE...

Esta relación no es cosmética: compartir código base con MATA implica que DRATzarus hereda capacidades de persistencia, evasión y comunicación C2 maduras y probadas en campañas anteriores.

Técnicas Anti-Análisis
Evasión de entornos de debugging y sandbox

Anti-Debug: IsDebuggerPresent

DRATzarus implementa una comprobación anti-debug básica pero efectiva usando la función de la API de Windows: 

BOOL WINAPI IsDebuggerPresent(void);Windows API

Esta función devuelve TRUE si el proceso está siendo depurado. DRATzarus la llama durante la inicialización: si detecta un debugger, puede terminar su ejecución, dormir durante largos períodos o exhibir comportamiento benigno para engañar al analista.

💡 Bypass de IsDebuggerPresent Los analistas de malware suelen parchear esta llamada a 0 en memoria, o usar debuggers con plugins como ScyllaHide que engañan al malware haciéndole creer que no está siendo depurado.

Torisma — Implante de Reconocimiento

Torisma (MITRE S0678) es otro implante del arsenal Lazarus, usado habitualmente como herramienta de reconocimiento y segunda etapa tras el compromiso inicial. Su análisis revela técnicas de ofuscación sofisticadas:

CIFRADO C2
XOR + VEST-32
Las comunicaciones con el servidor de Command & Control están cifradas con una combinación de XOR y el cifrado por bloques VEST-32, dificultando el análisis de tráfico de red.
COMPRESIÓN
LZ4 Compression
El payload de Torisma está comprimido con LZ4, un algoritmo de compresión de alta velocidad. Esto reduce el tamaño del binario y dificulta la detección por firmas de antivirus.

💿
Vector de Entrega — Imagen ISO

La cadena de infección de Dream Job-2 utiliza una imagen ISO como vehículo de entrega. Este enfoque se volvió popular entre actores APT porque Windows 10/11 monta automáticamente los ISOs al hacer doble clic, y muchos controles de seguridad de email no inspeccionan su contenido.

Análisis del ISO en VirusTotal
Identificación del ejecutable embebido

El análisis del archivo ISO en VirusTotal permite identificar su contenido sin necesidad de ejecutarlo, obteniendo el hash del ejecutable embebido y su comportamiento documentado:

NOMBRE EN ISO
InternalViewer.exe
NOMBRE ORIGINAL
SumatraPDF.exe
El ejecutable fue renombrado desde su nombre original SumatraPDF.exe (un lector de PDF legítimo) a InternalViewer.exe para parecer una herramienta corporativa interna. Primera vez visto en la naturaleza: 2020-08-13 08:44:50 UTC.
3
UPX Packing
Ofuscación mediante empaquetado

El ejecutable está empaquetado con UPX (Ultimate Packer for eXecutables), uno de los packers más utilizados tanto por malware como por software legítimo. UPX comprime el binario original y lo descomprime en memoria durante la ejecución, lo que tiene dos efectos:

EfectoImpacto en el análisis
El tamaño del ejecutable se reduce significativamenteMenor tiempo de transferencia, menos sospechoso
Las firmas de antivirus no coinciden directamenteEvasión de detección basada en firmas estáticas
El código real solo es visible en memoria (dinámico)Requiere análisis dinámico o dumping de memoria
 
upx -d InternalViewer.exe -o InternalViewer_unpacked.exeUPX — Desempaquetar

Directorio Sospechoso — Análisis de Plantilla

El análisis del archivo de plantilla 17.dotm (asociado al documento señuelo) revela actividad de reconocimiento en el sistema de la víctima: 

\AppData\Local\Microsoft\NoticeDIRECTORIO SOSPECHOSO

Este directorio no es estándar en una instalación normal de Windows Office. El malware lo crea para almacenar datos y busca el archivo wsuser.db en su interior, probablemente como mecanismo de persistencia o recolección de datos.

Conclusiones

Dream Job-2 demuestra la madurez técnica del arsenal de Lazarus Group. La combinación de un RAT derivado de un framework maduro (MATA), técnicas anti-análisis múltiples, uso de ISO como vector y ejecutables UPX-packed rebautizados configura una cadena de ataque que maximiza las probabilidades de evasión en cada etapa.

✅ Lecciones de Malware Analysis El análisis estático con VirusTotal y la revisión de metadatos pueden revelar información crítica de atribución sin necesidad de ejecutar el malware. Las herramientas de unpacking como UPX -d son esenciales para la reversión de ejecutables empaquetados antes del análisis profundo.
🔴 Detección y Respuesta Monitorizar creación de directorios no estándar en AppData, ejecutables que llaman a IsDebuggerPresent, y presencia de UPX en ejecutables de nombre genérico son señales de alerta temprana para este tipo de campaña.
