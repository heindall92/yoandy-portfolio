📋Escenario

El equipo de seguridad ha recibido una alerta sobre actividad sospechosa en el endpoint de una analista llamada Susan. Se sospecha que el grupo de amenaza persistente avanzada RomCom ha comprometido el sistema a través de un archivo RAR malicioso. Como analista DFIR, se proporcionan artefactos de disco en formato VHDX para investigar el incidente de forma forense y reconstruir la cronología del ataque.

⚠ Threat Actor RomCom es un grupo APT de origen ruso conocido por combinar ransomware con espionaje, dirigido especialmente contra organizaciones gubernamentales, sanitarias y de defensa. Sus TTPs incluyen el uso de exploits zero-day en software ampliamente usado.

🔧Entorno y Preparación

El artefacto proporcionado es una imagen de disco virtual en formato VHDX (Virtual Hard Disk Extended), un formato de Microsoft que preserva la integridad del sistema de archivos NTFS original. El análisis se realiza sin modificar la evidencia original.

1
Verificación de Integridad
Validación SHA256 antes de cualquier análisis

Antes de comenzar el análisis, se verifica la integridad del archivo con su hash SHA256 para asegurar que la evidencia no ha sido alterada durante la cadena de custodia.

Get-FileHash .\romcom.vhdx -Algorithm SHA256PowerShell
2
Montaje del VHDX
Acceso a los artefactos del sistema de archivos

El VHDX se monta en modo solo lectura para acceder al sistema de archivos NTFS. Los artefactos clave de NTFS que se extraen para el análisis son:

ArtefactoDescripciónInformación forense
$MFTMaster File TableRegistro de todos los archivos: nombres, timestamps, tamaños, atributos
$J ($UsnJrnl)USN Change JournalHistorial de cambios del sistema de archivos en tiempo real
3
Parsing con Eric Zimmerman Tools
MFTECmd + Timeline Explorer

Se utilizan las Eric Zimmerman Tools, el conjunto de herramientas forenses más robusto disponible para análisis NTFS en Windows. MFTECmd parsea el $MFT y genera un CSV que luego se carga en Timeline Explorer para análisis visual. 

MFTECmd.exe -f "D:\$MFT" --csv "C:\analysis" --csvf mft_output.csvCMD

🔍Investigación

Vector de Infección — CVE-2025-8088

El punto de entrada del ataque es un archivo RAR cuidadosamente nombrado para generar confianza en el objetivo. Al analizar la tabla maestra de archivos, se identifica el primer artefacto relevante: un archivo comprimido que explota la vulnerabilidad CVE-2025-8088, un path traversal en WinRAR que permite colocar archivos en directorios arbitrarios del sistema al extraer el archivo comprimido.

ARTEFACTO INICIAL
Pathology-Department-Research-Records.rar
Nombre diseñado para parecer legítimo en contexto sanitario/investigación

Análisis de Timestamps en el $MFT

Mediante el análisis cronológico en Timeline Explorer, se reconstruye la secuencia de eventos con precisión de segundos:

2025-09-02   08:13:50 UTC
Creación del archivo RAR malicioso
El archivo Pathology-Department-Research-Records.rar aparece en el sistema de archivos.
2025-09-02   08:14:04 UTC
Apertura y extracción del RAR
Susan abre el archivo. WinRAR procesa el contenido y explota CVE-2025-8088. Transcurren solo 14 segundos desde la creación hasta la apertura.
Extracción inmediata
Despliegue del backdoor
El path traversal coloca el ejecutable malicioso fuera del directorio de extracción esperado.
Post-extracción
Establecimiento de persistencia
Se crea un archivo LNK en la carpeta Startup para asegurar la ejecución en cada inicio del sistema.

Técnica de Engaño — Decoy Document

Para evitar levantar sospechas en el objetivo, el RAR malicioso incluye un archivo señuelo legítimo que se abre automáticamente. Esto es una técnica de ingeniería social clásica de los grupos APT: el usuario ve un documento real y no percibe ninguna anomalía.

DOCUMENTO SEÑUELO
Genotyping_Results_B57_Positive.pdf
Documento PDF de apariencia médica. Mientras Susan lee el PDF, el backdoor ya está instalado.

Backdoor y Persistencia

El análisis del $MFT revela la ubicación exacta del componente malicioso desplegado por CVE-2025-8088: 

C:\Users\Susan\AppData\Local\ApbxHelper.exeBACKDOOR PATH

El directorio AppData\Local es una ubicación habitual para malware en Windows, ya que no requiere privilegios elevados para escribir en él y pasa desapercibido ante usuarios no técnicos.

El mecanismo de persistencia utilizado es un archivo LNK colocado en la carpeta de inicio del sistema (Startup). Esto garantiza que el backdoor se ejecute automáticamente cada vez que Susan inicia sesión. 

C:\Users\Susan\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\RUTA DE PERSISTENCIA

📊Conclusiones

El caso RomCom demuestra la efectividad de los ataques dirigidos que combinan exploits de software legítimo (WinRAR) con técnicas de ingeniería social sofisticadas. El análisis forense de artefactos NTFS permitió reconstruir con precisión de segundos la cadena de infección completa.

✅ Lecciones Aprendidas El $MFT y el USN Journal son artefactos críticos para la forensia de Windows. Incluso cuando el atacante intenta limpiar sus huellas, estos artefactos preservan evidencia temporal que permite la reconstrucción del ataque. La aplicación de parches de software de terceros como WinRAR es tan importante como actualizar el sistema operativo.
🔴 Indicadores de Compromiso (IOC) Archivo RAR con nombre de temática médica/investigación + documento PDF señuelo + ejecutable en AppData\Local + LNK en Startup son una combinación característica de este grupo de amenaza.
