Archetype — MSSQL + SMB + WinPEAS Privesc

Máquina Windows que expone SMB con share abierto conteniendo credenciales de MSSQL. Se obtiene ejecución de comandos via xp_cmdshell, reverse shell con PowerShell, y escalada a Administrator mediante WinPEAS descubriendo credenciales en el historial de consola PowerShell.

Escenario

Archetype es una máquina Windows del Starting Point de HackTheBox con dirección IP 10.129.96.188. El objetivo simula un entorno corporativo donde un servidor MSSQL tiene habilitado el procedimiento almacenado xp_cmdshell, permitiendo ejecución de comandos del sistema.

El vector inicial consiste en acceder a un share SMB de manera anónima y recuperar un archivo de configuración que contiene credenciales en texto plano para el servicio SQL Server. Desde allí, la escalada aprovecha prácticas inseguras de historial de comandos en PowerShell.

Cadena de Ataque

Paso 1
Enumeración SMB Anónima
Listado de shares con acceso anónimo. Se descubre el share backups con el archivo prod.dtsConfig.
Paso 2
Credential Disclosure en DTS Config
El archivo de configuración SSIS expone credenciales MSSQL: ARCHETYPE\sql_svc:M3g4c0rp123.
Paso 3
Abuso de xp_cmdshell en MSSQL
Habilitación del procedimiento almacenado xp_cmdshell para ejecutar comandos del sistema operativo como sql_svc.
Paso 4
PowerShell Reverse Shell → User Flag
Descarga y ejecución de script PowerShell para obtener shell interactiva. Se obtiene la user flag del escritorio de sql_svc.
Paso 5
PSHistory → Credenciales de Administrator
WinPEAS revela el historial de comandos PowerShell con credenciales del Administrador: MEGACORP_4dm1n!!.
Paso 6
psexec.py → NT AUTHORITY\SYSTEM
Uso de impacket psexec con las credenciales del administrador para obtener shell como SYSTEM y recuperar la root flag.
